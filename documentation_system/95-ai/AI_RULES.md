## Dev Log (ОБЯЗАТЕЛЬНО — только при закрытии сессии)
- Сессию закрывает ТОЛЬКО владелец — он сам говорит когда
- AI не логирует сам по себе после каждой задачи
- AI генерирует команду логирования только когда владелец объявил закрытие сессии

## Dev Log (команда)
- Таблица dev_sessions в Supabase, API на Render
- Команда:
  curl -X POST https://insalon.onrender.com/dev-sessions \
    -H "Content-Type: application/json" \
    -d '{"date":"YYYY-MM-DD","feature":"...","category":"dev","duration_min":90,"tokens_approx":50000,"notes":"..."}'
- Категории: dev / design / analytics
- AI генерирует команду сам — не ждёт напоминания от владельца

## Обновление знаний (в момент появления)
- Если владелец поправил AI или уточнил правило — это акт фиксации знания
- AI сразу предлагает внести изменение в базу знаний через bash-команду
- Не ждать конца сессии — знание фиксируется в момент когда оно появилось
- Любая поправка от владельца = потенциальное обновление документации

## Документация (ОБЯЗАТЕЛЬНО в конце каждой сессии)
- AI сам генерирует bash-команды для обновления файлов
- Владелец только запускает команду в терминале
- Обновлять только те файлы которые реально изменились
- Runtime-файлы: CURRENT_STATE.md, ACTIVE_SPRINT.md, KNOWN_ISSUES.md
- После обновления: git add -A && git commit -m "docs: session YYYY-MM-DD" && git push

## Применение изменений в файлы
AI-оператор ВСЕГДА применяет изменения в файл через bash/str_replace напрямую.
НИКОГДА не отдаёт куски кода с инструкцией «вставь после строки X».
Файл готов к деплою — пользователь только копирует в репо.
