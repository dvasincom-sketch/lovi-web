Содержит ключевую бизнес-логику аналитических расчётов:
- Расчёт выручки из `transactions` с фильтрацией по проекту salon.
- Агрегация KPI за скользящие 30 дней.
- Применение правила двойных смен (выручка / 2) при расчёте эффективности.
- Абонементы: для P&L в день продажи, для мастера — в день использования (price × 0.7).
- Формирование P&L с исключением личных расходов.
- CM/LH: (выручка - переменные расходы) / оплаченные часы.
- Логика сверки выплат через сопоставление payroll и personal_transactions (expense_category='salary', project='salon').
