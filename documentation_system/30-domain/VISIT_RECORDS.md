## Назначение
Таблица `visit_records` фиксирует выходы мастеров под запись (парные программы). Содержит: date, staff_name, service_title, visit_pay, notes.

## Тариф выхода под запись
| Duration (сек) | Минуты | Оплата |
|---|---|---|
| < 6300 | < 105 мин | 1 500 ₽ |
| 6300–8400 | 105–140 мин | 2 000 ₽ |
| > 8400 | > 140 мин | 3 000 ₽ |

Пример: «Экспресс» для двоих (6000 сек = 100 мин) → 1 500 ₽.

## Логика заполнения
- Исторические данные берутся из `notes` таблицы payroll.
- Данные за март 2026 внесены вручную (пример: Анастасия: 04(1500+2000), 09(5000) ...).
- Связь с парными программами: если service_cost < price_min * 0.5 — запись не считается парной (второй клиент не пришёл). Исключение: оплата сертификатом может дать 50% при полной явке — проверять вручную.

## API
- GET /analytics/visit-records/{year}/{month} — возвращает выходы под запись по датам.
- Формат ответа: `{"visit_days": { "4": [{"staff_name": "...", "visit_pay": ...}] } }`
