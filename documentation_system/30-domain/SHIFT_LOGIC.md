## Модель смен
Смены мастеров хранятся в таблице `shifts` (заполняется вручную из-за недоступности YCLIENTS API).

| Поле | Описание |
|------|----------|
| date | Дата смены |
| staff_name | Имя мастера |
| shift_pay | 5000 за обычную смену, 0 для выхода под запись |
| is_double_shift | true, если 2 мастера в день |
| is_visit_only | true, если выход под запись (нет смены) |
| notes | Комментарий |

## Типы выходов
- **Обычная смена**: мастер работает полную смену, оплата 5000.
- **Двойная смена**: два мастера в один день. Выручка дня делится поровну (при расчёте эффективности). Оба получают по 5000 за смену.
- **Выход под запись**: мастер приходит только на конкретную услугу (обычно парная программа). shift_pay = 0, оплата через visit_pay (см. VISIT_RECORDS.md).
- **Особый случай — Мария**: работает только под запись, смен нет (shifts=0, shift_pay=0). В YCLIENTS значится как "Александра К.". Авансы/выплаты переводом. Записи в payroll создаются вручную, автосинхронизация не применяется.

## Правила определения выхода под запись
- 1 мастер в смене + парная программа в этот день → второй мастер вышел под запись.
- 2 мастера в смене + парная программа → парная входит в смену, выход под запись не считается.
- Исторические данные парсятся из поля `notes` таблицы payroll.
- Данные за март 2026 заполнены вручную.

## Отображение в календаре (дашборд)
- Серая плашка (`bg-secondary`) — смена (из `shifts`)
- Розовая плашка (`bg-pink-lt`) — выход под запись (из `visit_records` или `couple-programs`)
- `? неизвестен` — есть парная, но мастер не определён.
- 2 мастера в смене → парные не показываем.
- Мастер из records не должен совпадать с мастером смены (проверка `shiftNames.includes`).
