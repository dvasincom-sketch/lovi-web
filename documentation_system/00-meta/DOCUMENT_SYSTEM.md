Структура документации, правила классификации и поддержки.

## Типы документов
- **Canonical** — стабильный источник истины (бизнес-логика, архитектура, API, схемы). Не содержит временных заметок, задач спринта или отладочной болтовни.
- **Runtime** — оперативное состояние (текущий спринт, блокеры, баги, статус миграций). Может меняться ежедневно.
- **Deprecated** — устаревшие системы, перемещены в /archive/.

## Правила поддержки
1. Любая бизнес-логика живёт в одном месте (например, формулы payroll только в PAYROLL_RULES.md).
2. Доменные инварианты, пограничные случаи и исключения фиксируются в соответствующих доменных файлах.
3. Каждый файл начинается с заголовка: Status, Owner, Last Updated, Source of Truth, Related Documents.
4. Изменения вносятся через AI или разработчиком, с обновлением даты.
